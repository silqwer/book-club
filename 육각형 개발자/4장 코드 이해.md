# 코드 이해 도구

코드를 변경할 때 2단계

- 코드 이해
- 코드 수정

코드를 읽고 나서 논리 구조를 파악하고 실행 흐름에 따라 상태 변경을 추적해야 한다.

코드를 이해하는 시간을 줄이기 위한 2가지 역량

- 이해
- 쉬운 코드를 작성

코드 분석에 도움이 되는 도구

- 시각화
- 코드 출력
- 스크래치 리팩터링

## 스크래치 리팩터링

코드 분석을 목적으로 하는 리팩터링이다. 리팩터링을 하려면 코드 구조나 의미를 이해하는 과정을 거쳐야 하는데 스크래치 리팩터링은 바로 여기에 집중한다.

리팩터링한 결과는 반영하지 않고 다시 원래대로 되돌린다.

## 함께 모여 보기

코드를 함께 보면 다음과 같은 이점이 있다.

- 코드에 대한 지식이 여러 명에게 쌓인다.
- 서로의 지식을 주고받으며 부족한 부분을 채울 수 있다.
- 잘못 이해하는 것을 줄일 수 있다.

## 이해를 돕는 몇 가지 코드 작성법

- 이름
- 중첩 if 최소화
- 변수 줄이기
- 값 변경 최소화하기
- 알맞은 파라미터 타입 사용하기
- 길지 않은 코드와 메서드 추출
- 추상화 수준 맞추기
